# Расширение анализа каналов цвета в пространстве OKLCH

## Функция fix-hue: улучшенная обработка каналов цвета

### Обзор

Функция `fix-hue` предназначена для коррекции и анализа цветовых каналов при переводе цвета в пространство OKLCH (Lightness, Chroma, Hue). Основная цель - обеспечить корректную работу с единицами измерения и ограничить изменения каналов в рамках их допустимых диапазонов.

### Ключевые улучшения

Переименовать `fix-hue` в `fix-channels` т.к. теперь валидируются и другие каналы.

#### 1. Валидация единиц измерения

**Проблема:** Некоторые операции могут возвращать значения без единиц измерения, когда они требуются.

**Решение:**
- Автоматическая проверка наличия единиц измерения в каждом канале
- Добавление единиц измерения по умолчанию при их отсутствии
- Предотвращение операций с некорректными единицами

```scss
// Пример валидации
@function validate-channel($value, $channel) {
  @if unitless($value) {
    @warn "Значение для канала #{$channel} должно иметь единицу измерения";
    @return $value + get-default-unit($channel);
  }
  @return $value;
}
```

#### 2. Ограничение диапазонов каналов

**Диапазоны OKLCH:**
- **Lightness (L):** 0% - 100% (относительная яркость)
- **Chroma (C):** 0 - 0.37 (насыщенность цвета)
- **Hue (H):** 0° - 360° (цветовой тон)

**Реализация ограничений:**

```scss
@function clamp-channel($value, $channel) {
  @if $channel == 'lightness' {
    @return clamp(0%, $value, 100%);
  } @else if $channel == 'chroma' {
    @return clamp(0, $value, 0.37);
  } @else if $channel == 'hue' {
    @return $value % 360deg;
  }
}
```

### Алгоритм работы fix-hue

#### Шаг 1: Анализ входных данных
1. Проверка формата входного цвета
2. Извлечение значений каналов L, C, H
3. Валидация единиц измерения

#### Шаг 2: Коррекция каналов
1. Применение ограничений диапазона
2. Нормализация значений
3. Добавление недостающих единиц измерения

#### Шаг 3: Фиксация оттенка
1. Анализ текущего значения hue
2. Коррекция для предотвращения нежелательных сдвигов
3. Обеспечение стабильности цветового тона

### Практическое применение

#### Базовое использование
```scss
$color: oklch(70% 0.2 120deg);
$fixed: fix-hue($color);
```

#### Продвинутое использование с параметрами
```scss
$color: oklch(85% 0.35 240deg);
$fixed: fix-hue(
  $color,
  $preserve-lightness: true,
  $clamp-chroma: true,
  $normalize-hue: true
);
```

### Обработка крайних случаев

#### 1. Нулевые значения
- Lightness: 0% (черный) или 100% (белый)
- Chroma: 0 (серая шкала)
- Hue: игнорируется при Chroma = 0

#### 2. Граничные значения
- Максимальная Chroma зависит от Lightness и Hue
- Автоматическая коррекция при выходе за пределы цветового гамута

#### 3. Некорректные единицы
```scss
// Неправильно
$invalid: oklch(70 0.2 120); // отсутствуют единицы

// Правильно после fix-hue
$corrected: fix-hue($invalid); // oklch(70% 0.2 120deg)
```
