# Алгоритмы работы функций цветовой системы

## Обзор архитектуры

Модуль `_colors.scss` реализует систему управления цветами на основе цветового пространства OKLCH. Основные возможности:
- **Интерполяция оттенков** — генерация светлых/темных вариантов базового цвета
- **Автоматический контраст** — определение цвета текста по светлоте фона
- **CSS-переменные** — генерация именованных переменных для дизайн-системы
- **Поиск по палитре** — определение ближайшего оттенка по взвешенному расстоянию

---

## Внутренние функции (приватные)

### `_get-oklch-params($color)`

**Назначение:** Преобразует цвет в пространство OKLCH и извлекает его параметры.

**Алгоритм:**
1. Конвертирует входной цвет в OKLCH через `color.to-space()`
2. Извлекает три канала: `lightness`, `chroma`, `hue`
3. Возвращает map с цветом и отдельными параметрами

**Возвращаемое значение:**
```scss
(
  'color': <oklch-color>,
  'lightness': <percentage>,
  'chroma': <number>,
  'hue': <angle>
)
```

---

### `_get-map-by-shade($color)`

**Назначение:** Находит ближайший оттенок в палитре для заданного цвета.

**Алгоритм (взвешенное евклидово расстояние в OKLCH):**

1. **Конвертация цвета** — входной цвет преобразуется в OKLCH
2. **Итерация по палитрам** — перебор всех палитр (`$_base-colors-map`)
3. **Итерация по оттенкам** — для каждой палитры перебор оттенков (50, 100, 200...950)
4. **Вычисление разниц по каналам:**
   - **Hue (круговая разница):** `|Δhue|` с корректировкой через 180° (учитывает цикличность)
   - **Chroma (линейная разница):** простое вычитание
   - **Lightness (нормализованная):** разница делится на 1%
5. **Взвешенное расстояние:**
   ```
   distance = √[(ΔL × 1.0)² + (ΔH × 1.5)² + (ΔC × 2.0)²]
   ```
   Веса: lightness=1.0, hue=1.5, chroma=2.0
6. **Поиск минимума** — отслеживание ближайшего оттенка

**Возвращаемое значение:**
```scss
(
  'color': <closest-color>,
  'shade': <shade-number>,    // например, 500, 600
  'scope': <palette-name>      // например, 'red', 'sky'
)
```

---

### `_get-interpolation-shade($scope, $shade)`

**Назначение:** Получает параметры интерполяции из карты `$_interpolation-map`.

**Алгоритм:**
1. Извлекает карту палитры по имени `$scope`
2. Возвращает параметры для конкретного оттенка `$shade`

**Структура параметров:**
```scss
(
  'lightness': <adjustment>,  // смещение светлоты
  'chroma': <adjustment>,    // смещение насыщенности
  'hue': <adjustment>        // смещение оттенка
)
```

---

### `_shade-color($color, $step, $alpha)`

**Назначение:** Вычисляет оттенок цвета на основе параметров интерполяции.

**Алгоритм:**
1. **Конвертация в OKLCH** — `color.to-space($color, oklch)`
2. **Коррекция каналов** — вызов `fix-channels()` для валидации
3. **Определение палитры** — `_get-map-by-shade()` находит ближайшую палитру
4. **Получение параметров** — `_get-interpolation-shade()` даёт смещения для оттенка
5. **Применение смещений** — `color.adjust()` модифицирует LCH каналы
6. **Установка прозрачности** — `color.change()` применяет альфа-канал

**Пример:** Для `$step=600` применяются параметры затемнения, для `$step=400` — осветления.

---

### `_process-color($color, $alpha, $shade-effect-name, $shade-step, $return-var)`

**Назначение:** Единая точка обработки всех цветовых операций.

**Алгоритм (4 этапа):**

**Этап 1: Определение базового имени**
- Поиск в `$_base-colors-reversed` (значение → имя)
- Поиск в `$variables-map-colors` (для ранее сгенерированных)
- Извлечение из CSS-переменной (`var(--color-{name})`)

**Этап 2: Вычисление цвета**
- Если `$shade-effect-name == 'contrast'` → вызов `_contrast()`
- Иначе → вызов `_shade-color()` с интерполяцией

**Этап 3: Регистрация переменной**
- Генерация имени переменной через `css-vars` helpers
- Сохранение в `$variables-map-colors` через `_set-color-in-variables-map()`

**Этап 4: Возврат результата**
- Если `$return-var: true` → возвращает `var(--color-{name})`
- Иначе → возвращает цветовое значение OKLCH

---

### `_contrast($color, $alpha)`

**Назначение:** Определяет контрастный цвет текста по светлоте фона.

**Алгоритм:**
1. **Получение цветов контента** — `content-dark` (#222233) и `content-light` (#ffffff)
2. **Обработка CSS-переменных** — если передана `var(--color-*)`, ищет в `$variables-map-colors`
3. **Конвертация в OKLCH** — для извлечения канала `lightness`
4. **Сравнение с порогом** — `$_contrast-threshold: 56%`
   - `lightness > 56%` → темный текст (content-dark)
   - `lightness ≤ 56%` → светлый текст (content-light)
5. **Применение прозрачности** — если `$alpha < 1`

**Возвращаемое значение:**
```scss
(
  'shade-effect': 'dark' | 'light',
  'color': <contrast-color>
)
```

---

### `_set-color-in-variables-map($name, $color)`

**Назначение:** Дедупликация — предотвращает повторную регистрацию одинаковых переменных.

**Алгоритм:**
1. Проверяет наличие `$name` в `$variables-map-colors`
2. Если отсутствует — добавляет через `map.set()` с флагом `!global`
3. Возвращает имя переменной

---

### `fix-channels($color, $preserve-lightness, $clamp-chroma, $normalize-hue)`

**Назначение:** Валидация и коррекция всех каналов OKLCH.

**Алгоритм по каналам:**

| Канал | Валидация | Ограничение | Значение по умолчанию |
|-------|-----------|-------------|----------------------|
| `lightness` | Должен иметь `%` | 0% – 100% | 50% |
| `chroma` | Без единиц | 0 – 0.37 | 0.1 |
| `hue` | Должен иметь угол | 0° – 360° (нормализация) | 0° |

**Нормализация hue:**
- `deg`: `$value % 360deg`
- `rad`: `($value % 2π) * 1rad`
- `turn`: `($value % 1) * 1turn`

### Вспомогательные функции

| Функция | Назначение |
|---------|------------|
| `_get-default-unit($channel)` | Возвращает единицу по умолчанию: lightness→%, chroma→'', hue→deg |
| `_validate-channel($value, $channel)` | Проверяет тип и единицы, конвертирует безразмерные значения |
| `_clamp-channel($value, $channel)` | Ограничивает значение допустимым диапазоном |

## Публичные API-функции

### Функции возвращающие CSS-переменные (`var(--color-*)`)

Все функции передают параметры в `_process-color()` с `$return-var: true`:

| Функция | Оттенок | Эффект | Описание |
|---------|---------|--------|----------|
| `base($color, $alpha)` | 500 | base | Базовый цвет |
| `light-10($color, $alpha)` | 400 | light-10 | Светлее на 1 уровень |
| `light-20($color, $alpha)` | 300 | light-20 | Светлее на 2 уровня |
| `light-30($color, $alpha)` | 200 | light-30 | Светлее на 3 уровня |
| `light-40($color, $alpha)` | 100 | light-40 | Светлее на 4 уровня |
| `light-50($color, $alpha)` | 50 | light-50 | Максимально светлый |
| `dark-10($color, $alpha)` | 600 | dark-10 | Темнее на 1 уровень |
| `dark-20($color, $alpha)` | 700 | dark-20 | Темнее на 2 уровня |
| `dark-30($color, $alpha)` | 800 | dark-30 | Темнее на 3 уровня |
| `dark-40($color, $alpha)` | 900 | dark-40 | Темнее на 4 уровня |
| `dark-50($color, $alpha)` | 950 | dark-50 | Максимально темный |
| `content($color, $alpha)` | — | contrast | Контрастный цвет текста |

### Функции возвращающие цветовые значения (OKLCH)

Те же функции с суффиксом `-value` возвращают цвета вместо CSS-переменных:
- `base-value()`, `light-10-value()` ... `dark-50-value()`, `content-value()`

Параметр `$return-var: false` в `_process-color()`.

## Карты данных

### `$_base-colors`
Импорт из `hex-palette` — базовые цвета (имя → hex-значение).

### `$_base-colors-map`
Импорт из `palette-map` — палитры с оттенками (палитра → {оттенок: цвет}).

### `$_interpolation-map`
Импорт из `interpolation-map` — параметры смещения для каждого оттенка.

### `$_base-colors-reversed`
Генерируется автоматически — обратная карта (значение → имя) для быстрого поиска.

### `$variables-map-colors`
Глобальная карта всех сгенерированных CSS-переменных (имя → OKLCH-значение).

---

## Константы

| Константа | Значение | Назначение |
|-----------|----------|------------|
| `$_contrast-threshold` | 56% | Порог светлоты для переключения контраста |
| `$_distance-weights` | (L:1.0, H:1.5, C:2.0) | Веса для расчёта расстояния в OKLCH |

## Поток данных при обработке цвета

```
Вход: $color (hex/name/var)
  ↓
[_process-color]
  ├── Определение базового имени
  │     ├── $_base-colors-reversed
  │     ├── $variables-map-colors
  │     └── Парсинг var(--color-*)
  │
  ├── Вычисление результата
  │     ├── contrast → [_contrast]
  │     └── shade → [_shade-color]
  │           ├── color.to-space(oklch)
  │           ├── fix-channels
  │           ├── _get-map-by-shade
  │           │     └── Взвешенное расстояние OKLCH
  │           ├── _get-interpolation-shade
  │           └── color.adjust + color.change
  │
  ├── Регистрация в $variables-map-colors
  │     └── _set-color-in-variables-map
  │
  └── Возврат
        ├── var(--color-*) или
        └── OKLCH значение
```
