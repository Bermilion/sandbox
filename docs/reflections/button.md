# Архитектура компонента Button

## Обзор

Компонент кнопки Flux (`stubs/resources/views/flux/button/index.blade.php`) — это универсальная реализация кнопки с множеством вариантов для Laravel Blade с поддержкой Livewire. Он предоставляет гибкий API для создания кнопок с иконками, состояниями загрузки, горячими клавишами и различными визуальными вариантами.

## API компонента

### Параметры (Props)

| Параметр | Тип | По умолчанию | Описание |
|----------|-----|--------------|----------|
| `variant` | string | `'outline'` | Визуальный стиль: `primary`, `filled`, `outline`, `danger`, `ghost`, `subtle` |
| `size` | string | `'base'` | Размер кнопки: `base`, `sm`, `xs` |
| `type` | string | `'button'` | Тип HTML-кнопки: `button`, `submit`, `reset` |
| `icon` / `iconLeading` | string/null | `null` | Иконка слева (поддерживается алиас) |
| `iconTrailing` | string/null | `null` | Иконка справа |
| `iconVariant` | string/null | `null` | Вариант размера иконки: `micro`, `mini` (автоопределение) |
| `loading` | bool/null | `null` | Показать спиннер загрузки (автоопределение) |
| `square` | bool/null | `null` | Принудительная квадратная форма (автоопределение по пустому слоту) |
| `inset` | string/null | `null` | Применить позиционирование с отрицательным отступом |
| `kbd` | string/null | `null` | Отобразить бейдж с горячей клавишей |

### Иконки через атрибуты

Иконки также можно задавать через HTML-атрибуты:
- `icon:leading` → привязывается к `iconLeading`
- `icon:trailing` → привязывается к `iconTrailing`
- `icon:variant` → привязывается к `iconVariant`

## Слои архитектуры

### 1. Извлечение и настройка иконок (Строки 1-20)

```php
@php $iconTrailing = $iconTrailing ??= $attributes->pluck('icon:trailing'); @endphp
@php $iconLeading = $iconLeading ??= $attributes->pluck('icon:leading'); @endphp
@php $iconVariant = $iconVariant ??= $attributes->pluck('icon:variant'); @endphp
```

Сначала иконки извлекаются из атрибутов, затем параметр `icon` алиасится к `iconLeading`.

### 2. Логика авто-конфигурации (Строки 22-33)

**Определение квадратной формы:**
- Кнопка автоматически становится квадратной, если `$slot->isEmpty()` (кнопки только с иконкой)

**Автоопределение варианта иконки:**
- Размер `xs` → вариант `micro`
- Квадратные кнопки → вариант `mini` (более крупные иконки для кнопок только с иконкой)
- По умолчанию → вариант `micro`

**Размеры классов иконки:**
- Иконки outline в квадратных non-xs кнопках получают `size-5`
- Все остальные получают `size-4`

### 3. Определение состояния загрузки (Строки 35-50)

Состояние загрузки автоматически включается, когда:
1. Тип кнопки `submit` И не отключена при рендере
2. Есть атрибут `wire:click` (Livewire) И это не метод `$js.`

Для loading-кнопок не-submit типа компонент автоматически добавляет:
- Привязку атрибута `wire:loading.attr="data-flux-loading"`
- `wire:target`, привязанный к методу wire:click (предотвращает перекрестное срабатывание кнопок)

### 4. Конструирование CSS-классов (Строки 52-117)

Используется билдер `Flux::classes()` для создания Tailwind-классов:

**Базовые классы:**
- Расположение: `relative items-center font-medium justify-center gap-2 whitespace-nowrap`
- Состояния disabled: opacity, cursor, pointer-events

**Варианты размеров:**
| Размер | Высота | Текст | Скругление | Отступы (не-квадрат) |
|--------|--------|-------|------------|---------------------|
| `base` | `h-10` | `text-sm` | `rounded-lg` | `px-4` |
| `sm` | `h-8` | `text-sm` | `rounded-md` | `px-3` |
| `xs` | `h-6` | `text-xs` | `rounded-md` | `px-2` |

**Визуальные варианты:**

| Вариант | Фон | Текст | Граница | Тень |
|---------|-----|-------|---------|------|
| `primary` | Акцентный цвет с hover | Акцентный foreground | `border-black/10` | Внутреннее белое свечение |
| `filled` | Zinc темный/светлый с прозрачностью | Zinc/white | — | — |
| `outline` | White/zinc-700 | Zinc/white | Zinc с нижним акцентом | `shadow-xs` |
| `danger` | Red-500/600 | White | — | Красное внутреннее свечение |
| `ghost` | Прозрачный | Zinc/white | — | — |
| `subtle` | Прозрачный | Zinc-400 → hover zinc/white | — | — |

**Позиционирование inset:**
Классы отрицательных отступов применяются по сторонам в зависимости от размера:
- `base`: `-mt-2.5`, `-me-4/-2.5`, `-mb-2.5/-3`, `-ms-4/-2.5`
- `sm`: `-mt-1.5`, `-me-3/-1.5`, `-mb-1.5`, `-ms-3/-1.5`
- `xs`: `-mt-1`, `-me-2/-1`, `-mb-1`, `-ms-2/-1`

**Обработка границ групп кнопок:**
Специальные CSS-селекторы управляют границами между кнопками в группе:
- `outline`: Убирает левые границы у сгруппированных кнопок, сохраняет у первой
- `filled`: Правые границы между кнопками
- `danger`: Правые границы с красной заливкой
- `primary`: Сложная обработка границ в темном режиме

**Классы состояния загрузки:**
- `*:transition-opacity` на дочерних элементах
- Переходы прозрачности для индикатора загрузки
- Отключение pointer-events во время загрузки

### 5. Атрибут группы (Строки 119-122)

```php
$attributes = $attributes->merge([
    'data-flux-group-target' => ! in_array($variant, ['subtle', 'ghost']),
]);
```

Освобождает варианты `subtle` и `ghost` от переопределения скругления границ в группах кнопок.

### 6. Структура шаблона (Строки 125-158)

```
flux:with-tooltip (обертка)
  └── flux:button-or-link (семантический элемент)
      ├── Индикатор загрузки (абсолютное центрирование, условный)
      ├── Ведущая иконка (flux:icon или сырой HTML)
      ├── Содержимое слота (обернуто в span если loading)
      ├── Бейдж горячей клавиши (kbd)
      └── Трейлинг-иконка (flux:icon или сырой HTML)
```

## Ключевые архитектурные решения

1. **Паттерн Flux::classes()**: Использует fluent API для конструирования условных Tailwind-классов вместо конкатенации строк.

2. **Авто-определение**: Минимизирует API, выводя поведение из контекста:
   - Квадратная форма из содержимого
   - Загрузка из submit/wire:click
   - Размер иконки из контекста

3. **Интеграция с Livewire**: Полноценная поддержка `wire:click` и `wire:loading` с автоматическим scoping через `wire:target`.

4. **Извлечение атрибутов**: Позволяет задавать иконки через HTML-атрибуты для более чистого синтаксиса Blade: `<flux:button icon:leading="plus">`.

5. **Стили для групп**: Сложные CSS-селекторы обрабатывают границы групп кнопок без явной конфигурации группы.

6. **Обертка тултипа**: Все кнопки обернуты в `flux:with-tooltip` для единообразного поведения тултипов.

## Зависимости

- `Flux::classes()` - Утилита для построения классов
- `Flux::applyInset()` - Генератор отрицательных отступов
- `flux:with-tooltip` - Компонент-обертка тултипа
- `flux:button-or-link` - Компонент семантического элемента (кнопка vs ссылка)
- `flux:icon` - Компонент рендеринга иконок
