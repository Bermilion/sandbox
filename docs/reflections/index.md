# Архитектура проекта Flux

## 1. Общая информация

### Назначение проекта
Flux - это библиотека UI-компонентов для приложений Livewire, построенная на основе Tailwind CSS. Проект предоставляет набор готовых, легко настраиваемых компонентов для ускорения разработки пользовательских интерфейсов в Laravel-приложениях.

### Технологический стек
- PHP: ^8.1
- Laravel: v10.0+
- Livewire: v3.5.19+
- Tailwind CSS: v4.0+
- Symfony Console
- Laravel Prompts

### Системные требования
- PHP 8.1 или выше
- Laravel v10.0 или выше
- Livewire v3.5.19 или выше
- Tailwind CSS v4.0 или выше

## 2. Структура проекта

### Основные директории
- `src/` - основной код библиотеки
- `docs/` - документация
- `dist/` - скомпилированные ассеты
- `stubs/` - шаблоны для публикации компонентов
- `ia/` - вспомогательные файлы для архитектуры

### Ключевые файлы
- `composer.json` - файл зависимостей и автозагрузки
- `src/FluxServiceProvider.php` - сервис-провайдер Laravel
- `src/Flux.php` - основной класс библиотеки

### Схема namespace и автозагрузки
Проект использует PSR-4 автозагрузку с namespace `Flux\`, который маппится на директорию `src/`. Все классы библиотеки находятся в этом namespace.

## 3. Архитектурные решения

### Паттерны проектирования
- Service Provider - для интеграции с Laravel
- Singleton - для управления состоянием через `FluxManager`
- Facade - через алиас `Flux` в Laravel
- Concerns - для вынесения общего функционала (например, `InteractsWithComponents`)

### Принципы организации кода
- SOLID - соблюдение принципов объектно-ориентированного проектирования
- DRY - переиспользование кода через трейты и компоненты
- KISS - простота реализации компонентов

### Обработка ошибок и логирование
- Используются стандартные механизмы Laravel
- Ошибки валидации обрабатываются через Livewire
- Логирование не реализовано явно, полагается на инфраструктуру Laravel

## 4. Модули и компоненты

### Развернутое описание всех сущностей

#### AssetManager
Класс для управления ассетами Flux. Регистрирует Blade-директивы `@fluxAppearance` и `@fluxScripts`, а также маршруты для обслуживания JavaScript и CSS файлов. Обеспечивает кэширование ассетов и поддержку как бесплатной, так и Pro-версии библиотеки.

#### ClassBuilder
Простой строитель CSS-классов, реализующий интерфейс Stringable. Использует Laravel's Arr::toCssClasses для объединения классов и устранения дубликатов. Позволяет цепочечное добавление классов через метод add().

#### FluxManager
Центральный класс управления состоянием Flux. Реализует паттерн Singleton и предоставляет методы для работы с ассетами, компонентами и стилями. Содержит логику для проверки активации Pro-версии, управления классами, работы с атрибутами компонентов и взаимодействия с компонентами через трейт InteractsWithComponents.

#### FluxTagCompiler
Компилятор тегов Flux, расширяющий ComponentTagCompiler Laravel. Преобразует теги `flux:component` в вызовы компонентов Laravel. Поддерживает как открывающие, так и самозакрывающиеся теги. Интегрируется с Blade-компилятором для обработки синтаксиса Flux.

#### DateRange
Класс для работы с диапазонами дат, расширяющий CarbonPeriod. Предоставляет удобные статические методы для создания предустановленных диапазонов (сегодня, вчера, эта неделя и т.д.). Используется для фильтрации данных по временным интервалам.

#### DateRangePreset
Перечисление (enum) с предустановленными диапазонами дат. Содержит константы для различных временных интервалов (день, неделя, месяц, квартал, год) и методы для получения соответствующих дат и локализованных меток.

#### DateRangeSynth
Сериализатор для работы с диапазонами дат в Livewire. Реализует механизм синтеза/десериализации объектов DateRange для передачи между клиентом и сервером. Обеспечивает корректную передачу состояния диапазонов дат через Livewire.

#### Flux
Фасад для доступа к функциональности Flux. Предоставляет удобный статический интерфейс для вызова методов FluxManager через сервис-контейнер Laravel.


#### FluxServiceProvider
Сервис-провайдер Laravel для регистрации компонентов Flux. Регистрирует синглтоны, макросы, компилятор тегов, команды Artisan и настраивает пути к компонентам. Является точкой входа для интеграции Flux с приложением Laravel.

#### InteractsWithComponents
Трейт, добавляющий функциональность для взаимодействия с компонентами. Предоставляет методы для работы с модальными окнами (modal-show, modal-close) и тостами (toast-show). Реализует удобные интерфейсы для вызова этих компонентов из кода.

#### ActivateCommand
Команда Artisan для активации Pro-версии Flux. Позволяет пользователю ввести email и ключ лицензии, добавляет их в auth.json Composer и устанавливает пакет livewire/flux-pro. Обеспечивает простой процесс активации платной версии библиотеки.

#### IconCommand
Команда Artisan для импорта иконок из Lucide. Позволяет пользователю указать имя иконки, загружает SVG-файл с GitHub и создает Blade-компонент в ресурсах приложения. Обеспечивает расширяемость библиотеки иконок.

#### PublishCommand
Команда Artisan для публикации компонентов Flux в ресурсы приложения. Позволяет пользователю выбрать один или несколько компонентов для публикации, что позволяет кастомизировать их внешний вид и поведение. Поддерживает как бесплатные, так и Pro-компоненты.

### Основные модули
- `AssetManager` - управление ассетами и регистрация Blade-директив
- `ClassBuilder` - построение CSS-классов с использованием Laravel's Arr::toCssClasses
- `FluxManager` - центральный менеджер состояния и координации компонентов
- `FluxTagCompiler` - компиляция тегов Flux в компоненты Laravel
- `DateRange` - работа с диапазонами дат и периодами времени
- `DateRangePreset` - предустановленные диапазоны дат (перечисление)
- `DateRangeSynth` - сериализация/десериализация диапазонов дат для Livewire
- `Flux` - фасад для доступа к функциональности Flux
- `FluxServiceProvider` - сервис-провайдер для регистрации компонентов и макросов
- `InteractsWithComponents` - трейт для взаимодействия с компонентами (модалки, тосты)
- `ActivateCommand` - команда Artisan для активации Pro-версии
- `IconCommand` - команда Artisan для импорта иконок из Lucide
- `PublishCommand` - команда Artisan для публикации компонентов в ресурсы приложения

### Взаимодействие между модулями
Модули взаимодействуют через сервис-контейнер Laravel. `FluxServiceProvider` регистрирует основные сервисы (`FluxManager`, `AssetManager`), которые затем могут быть использованы другими компонентами. Компоненты обмениваются данными через Livewire-события (например, модалки используют `dispatch('modal-show')`). `FluxTagCompiler` интегрируется с Blade-компилятором для обработки тегов `flux:`. Команды Artisan используют сервисы Flux через контейнер зависимостей.

### API и интерфейсы
- Blade-директивы: `@fluxAppearance` (управление темой), `@fluxScripts` (подключение скриптов)
- Компоненты: Использование тегов `flux:component` для вставки UI-компонентов
- Команды Artisan: `flux:activate` (активация Pro-версии), `flux:icon` (импорт иконок), `flux:publish` (публикация компонентов)
- PHP API: Фасад `Flux` для доступа к функциональности
- Livewire-события: `modal-show`, `modal-close`, `toast-show` для взаимодействия между компонентами
- Макросы: `modal()`, `toast()` для удобного взаимодействия с компонентами
- Сериализаторы: `DateRangeSynth` для работы с диапазонами дат в Livewire

### Миграции и сиды
Проект не предоставляет миграции или сиды, так как не требует собственной схемы базы данных.

## 6. Безопасность

### Аутентификация и авторизация
Безопасность делегирована основному приложению Laravel. Flux использует существующие механизмы аутентификации и авторизации Laravel.

### Защита от уязвимостей
- CSRF-защита через Livewire
- Экранирование вывода через Blade
- Валидация входных данных через Livewire

### Валидация данных
Валидация осуществляется на уровне компонентов Livewire с использованием встроенных правил валидации Laravel.

### Кэширование
