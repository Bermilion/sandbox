# Структура проекта

Проект состоит из двух репозиториев:

1. **chunker2i/base** — основной репозиторий разработки (дочерний)
2. **sandbox** — репозиторий для отладки и тестирования (родительский)

## Назначение репозиториев

- **sandbox** — это Laravel-песочница, сконфигурированная для разработки пакетов Chunker. Используется исключительно для отладки функциональности пакетов из `chunker2i`.
- **chunker2i/base** — основной пакет разработки. Весь код компонентов реализуется там.

## Техническая интеграция

Пакеты лежат в игнорируемой гитом папке `chunker2i`. Каждый пакет — отдельная папка и отдельный репозиторий.

Пакеты выкачиваются через git в папку `chunker2i`. Оттуда по стандарту PSR-4 в `composer.json` пространства имён пакетов добавляются в песочницу. Через файл `bootstrap/providers` сервис-провайдеры пакетов подключаются в песочницу для публикации ассетов.

Ассеты для сборки (SCSS и JS) импортируются в оглавления песочницы `app.scss` и `app.js`. Пути до подключаемых файлов определяются псевдонимами в `vite.config.js`.

## Правило работы

**Все изменения реализуются в `chunker2i/base`.** Корневая директория `sandbox` используется только для проверки работоспособности компонентов.
